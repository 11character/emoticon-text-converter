# Project Status: Emoticon Text Converter

이 문서는 프로젝트의 개발 과정과 현재 상태를 기록하는 대시보드입니다.

## 1. 프로젝트 개요 (Overview)
- **목적**: 타입 안전성을 갖춘 순수 자바스크립트 기반의 범용 이모티콘 변환 라이브러리 구축.
- **핵심 목표**: 실시간 HTML 변환, 정밀한 커서 위치 유지, 이벤트 기반 인터페이스 제공.

## 4. 핵심 기술 노트 (Technical Notes)
- **다국어 지원**: `data-i18n` 속성과 중앙 집중식 `TRANSLATIONS` 객체를 통해 데모 페이지의 실시간 언어 전환(영/한) 구현.
- **타겟 검증**: 브라우저마다 `contenteditable`의 줄바꿈 처리가 다른 점을 고려하여, 가장 안정적인 `div` 요소만 타겟으로 허용하도록 강제.
- **커서 관리**: `img`(이모티콘)와 `br`을 각각 1글자로 취급하는 논리적 위치 시스템.
- **더미 BR 처리**: `getLineInfos`에서 문서 맨 끝의 `<br>`을 감지하여 유령 줄(ghost line) 생성을 방지하고 시각적 줄 수와 논리적 줄 수를 일치시킴.
- **이벤트 안전성**: 사용자 입력과 내부 변환 작업을 분리하여 무한 루프 및 충돌 방지.
- **텍스트 보정**: 이모티콘 삽입 시 이미지 태그 길이를 실제 키워드 길이로 자동 치환.
- **데모 사이트**: `docs/` 폴더 내 정적 빌드 파일을 통해 GitHub Pages 서비스 제공.
- **보안 설계**: 
    - `drop` 이벤트 차단 및 `text/plain`만 수용하여 HTML 삽입 공격 방지.
    - `toHtml` 변환 시 속성 이스케이프(`&quot;`)를 적용하여 속성 주입 공격 차단.
- **그룹 권한**: `allowedGroups` 설정을 통해 사용자별 이모티콘 노출 권한을 유연하게 제어. $O(1)$ 해시맵 조회를 통해 대규모 키워드 맵에서도 안정적인 성능 보장.
- **읽기 전용 모드**: `contenteditable` 속성을 제어하여 브라우저 수준에서 입력을 원천 차단하며, `aria-readonly` 및 전용 CSS 클래스를 통해 접근성과 스타일링 확장성 확보.
- **Undo/Redo**: 원본 텍스트와 논리적 커서 위치를 스택에 저장하여 복구하며, 복구 시 실시간 옵션을 재적용하는 로직 구현.
- **API 설계**: `setOptions`를 통한 일괄 설정 외에도 빈번하게 변경되는 UI 상태값(크기, 플레이스홀더 등)에 대해 직관적인 전용 메소드를 제공하여 개발자 편의성 향상.
- **코드 구조**: 대규모 클래스의 가독성을 위해 Core, Option, Content, Keyword, History, Info, Internal 등 7개 섹션으로 메소드를 그룹화하여 관리.
